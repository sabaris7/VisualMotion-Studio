VisualMotion Studio Documentation

Welcome to the official documentation for VisualMotion Studio. This guide provides comprehensive information on how to use, configure, and troubleshoot the application.

Note: This documentation applies to version 1.0 and above.

Features

VisualMotion Studio is a powerful browser-based SVG animation tool designed for developers and designers. Key features include:

  • Timeline-based Animation: Intuitive keyframe timeline for precise control over SVG properties.
  • Property Interpolation: Animate transforms (X, Y, Scale, Rotation), opacity, colors (fill, stroke), and motion paths.
  • Custom Easings: Support for linear, quadratic, cubic, bounce, and custom bezier curve easings.
  • Path Editing: Direct on-canvas editing of SVG paths with node manipulation and lasso selection.
  • Export Options: Export your animations as React components, Vanilla JS bundles, or standalone Animated SVGs.
  • Live Preview: Real-time rendering engine to preview animations exactly as they will appear in production.

Usage Guide

Importing SVG

To start animating, paste your raw SVG code into the "Import SVG" modal or use the default placeholder. The studio will automatically parse the SVG and populate the layer tree.

Selecting Layers

Click on any element in the stage or select it from the left sidebar. You can select multiple layers by holding Shift or Ctrl/Cmd while clicking.

Adding Keyframes

  1. Select a layer.
  2. Move the playhead on the timeline to the desired time.
  3. In the Properties panel on the right, change a value (e.g., X Position).
  4. A keyframe will be automatically added at the current time.

Configuration

VisualMotion Studio runs entirely in the browser and requires minimal configuration. However, you can adjust project-level settings:

  • Artboard Size: Set the width and height of the canvas.
  • Background Color: Define the artboard background.
  • Duration: Set the total duration of the animation loop.
  • Playback Options: Toggle looping or yoyo (ping-pong) playback.

Permissions

VisualMotion Studio operates client-side and requires the following browser permissions:

  • Clipboard Access: Required to copy exported code (React/Vanilla JS) to your clipboard.
  • Local Storage (Optional): Used to save your current workspace state so you don't lose progress on refresh.

Troubleshooting

Animations not playing smoothly

Ensure hardware acceleration is enabled in your browser. Complex SVGs with many gradients or filters may cause performance drops. Try simplifying the SVG paths.

Exported code not working

If the Vanilla JS export isn't running, ensure the script is placed at the end of your <body> tag or wrapped in a DOMContentLoaded event listener.

FAQ

Can I import complex SVGs from Illustrator or Figma?

Yes, but it is recommended to optimize and minify your SVGs before importing. Remove unnecessary groups and metadata for the best experience.

Is my data sent to a server?

No. VisualMotion Studio processes everything locally in your browser. Your SVG data never leaves your machine.
